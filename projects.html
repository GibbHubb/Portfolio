AI Document Intelligence — Huracán Tech

RET-RAG

A full production RAG (Retrieval-Augmented Generation) platform built internally at Huracán Technical Advisory. Enables teams to upload complex PDF and technical documents, chunk and embed them semantically, then query the knowledge base through a natural language interface powered by LLMs.

  • End-to-end RAG pipeline: document ingestion → Docling parsing → chunking → ChromaDB vector storage → LLM retrieval
  • FastAPI ingestion service with automated document parsing for complex PDFs and structured data
  • Custom React/TypeScript web interface for upload, monitoring, and interactive document querying
  • Deployed via Docker Compose on Proxmox VE with GPU-accelerated VMs for scalable model hosting
  • Multi-tenant architecture supporting concurrent users across isolated document collections
RAG Architecture LLMs ChromaDB Docling FastAPI Python React TypeScript Docker Proxmox VE

Architecture

FastAPI backend handles ingestion and serves the retrieval API. Docling parses raw PDFs into structured text; documents are chunked and embedded into ChromaDB. The React frontend lets users upload files, track processing status, and query documents dynamically. GPU-accelerated Proxmox VMs handle model inference at scale.

Highlights

  • Semantic search across large document collections with sub-second retrieval
  • Automated ingestion pipeline handles complex, multi-page technical PDFs
  • Multi-tenant Docker Compose deployment on Proxmox with GPU passthrough

Finance Tracker

Finly

A personal finance tracker built end-to-end. Real-time transaction feeds push updates to all connected clients via WebSockets — no page refresh needed. Full multi-currency support converts amounts on-the-fly using live exchange rates.

  • Real-time WebSocket broadcast: new transactions appear instantly across all open tabs
  • Multi-currency: per-user base currency + live Frankfurter rates with 1-hour cache
  • JWT-authenticated WebSocket endpoint — tokens validated on handshake
  • Alembic-managed schema migrations; base_currency on users, currency on transactions
  • Zustand store with duplicate-safe pushTransaction for optimistic updates
FastAPI React TypeScript Zustand WebSockets PostgreSQL SQLAlchemy Alembic Python

Architecture

FastAPI backend with async route handlers. SQLAlchemy ORM with Alembic migrations. React/TypeScript frontend using Zustand for global state. WebSocket connection manager keeps a live list of connected clients and broadcasts on every write.

Highlights

  • Exchange rate service proxies Frankfurter API with in-memory TTL cache
  • useTransactionSocket hook with exponential-backoff reconnect
  • Currency selector in dashboard header; per-row currency badge on transactions

Fitness Platform

GymPal

A trainer-client fitness platform with live session broadcasting. Trainers start a session and push exercises to individual clients in real time — clients receive them on their phone even if the app is backgrounded, thanks to Expo push notifications.

  • Socket.io room-based routing: user:{id} for session invites, client_session:{id} for exercise pushes
  • Expo push notifications via expo-server-sdk — notifies clients when session starts or workout assigned
  • TV/group session broadcast: trainer registers a named session, TV screens join and receive workout updates
  • Graceful offline-first sync (G3 planned): workout data cached locally and synced on reconnect
  • JWT auth + token validation on app startup; last-route restoration across sessions
React Native Expo Socket.io Node.js Express PostgreSQL Push Notifications

Architecture

Express backend served over a raw HTTP server to support Socket.io alongside REST routes. Room-based socket design keeps trainer→client targeting clean without storing socket IDs. expo-server-sdk on the server handles token lookup and chunked push batching.

Highlights

  • start_client_session fires both a socket event AND a push notification
  • Stale session cleanup runs every hour (12h TTL); heartbeat broadcasts every 10s
  • Client screens listen for session_started, exercise_pushed, session_ended events

AI Restaurant App

Our Menu

A recipe and meal-planning app that combines a restaurant menu interface with an AI backend. Users can ask natural-language questions about the recipe library (RAG over pgvector), generate personalised meal plans, and scan receipts to automatically populate a shopping list.

  • RAG pipeline: recipe library embedded into pgvector, similarity search on user queries
  • LLM-powered meal plan generation based on dietary preferences and available ingredients
  • Receipt scanning → OCR → structured shopping list, merged with existing pantry data
  • React Native mobile client with offline-capable recipe browsing
React Native RAG pgvector LLM Node.js PostgreSQL OCR

Architecture

pgvector extension on PostgreSQL stores recipe embeddings. A similarity search retrieves relevant recipes before passing context to the LLM. Node.js backend orchestrates the retrieval-augmented generation pipeline.

Highlights

  • Vector similarity search on recipe descriptions for natural-language queries
  • Meal plan generation respects dietary constraints and weekly variety
  • Receipt scanning automates shopping list creation

AI Education Tool

AI Learning Path Generator

An AI-powered tool that generates personalised learning curricula. Users describe their goal and starting level; the app produces a structured, step-by-step learning path with resource suggestions, streamed live as the LLM generates the response.

  • Streaming LLM responses via server-sent events — output appears word-by-word
  • Progress persistence: completed steps saved across sessions
  • Adapts to user's current skill level and desired depth
  • Clean React frontend with progress tracking UI
Python React LLM Streaming SSE FastAPI

Architecture

FastAPI backend with streaming response endpoint using server-sent events. Frontend consumes the stream and renders tokens progressively. Session data persists completion state so users can resume where they left off.

Highlights

  • Progressive rendering makes responses feel instantaneous
  • Structured output: module → topic → resource hierarchy
  • Step-level completion tracking with visual progress bar
